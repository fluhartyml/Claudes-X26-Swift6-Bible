Reference/Assets.xcassets

  1. Asset catalog (Assets.xcassets)

  2. Reference: Xcode resource bundle

  3. What this file is

  4. Despite the file extension, Assets.xcassets is a directory bundle — a folder with structured subfolders Xcode treats as one document. Inside, each "asset set" (image set, color set, app icon set, etc.) is its own subfolder with a Contents.json describing variants. At build time, Xcode's asset catalog compiler turns it into a single optimized binary called Assets.car that ships in your app bundle.

  5. What goes in it

    • App icon — the icon for your app on Home Screen, Settings, Spotlight, App Store. Modern Xcode (15+) accepts a single 1024×1024 master and generates the size ladder automatically; older configurations want each size individually.
    • Accent color — the global tint color for your app. SwiftUI's .tint modifier picks this up automatically. Set once here, applies everywhere.
    • Image sets — raster images you reference by name. Each set can have 1×, 2×, 3× variants for different screen densities, plus separate light / dark / tinted versions for system appearance modes (iOS 18+ tinted icon support is in tinted variant).
    • Color sets — named colors with light/dark/high-contrast variants. Reference as Color("BrandPrimary") in SwiftUI.
    • Symbol sets — for custom SF Symbols you've created.

  7. Less-common asset types

    • Data sets — for arbitrary binary blobs (model files, JSON configs); access via NSDataAsset.
    • AR resources — reference images and reference objects for ARKit.
    • Sticker packs — for Messages app sticker extensions.
    • BrandAssets — for tvOS top-shelf images.
    • App Clip backgrounds — for App Clip card art.

  9. iOS 18+ tinted-icon variants

  10. iOS 18 added user-tintable home-screen icons. To opt your app icon into that, supply a tinted variant alongside the regular icon — a flat single-color version of your icon art. The system tints it to whatever color the user picks. If you don't ship one, the system auto-generates a (usually worse) tinted version from your full-color icon.

  11. How you reference assets in code

  12. // SwiftUI
Image("hero-photo")              // image set named "hero-photo"
Color("BrandPrimary")            // color set named "BrandPrimary"

// UIKit
UIImage(named: "hero-photo")
UIColor(named: "BrandPrimary")

  13. Watch out for

    • Spelling matters. The compiler can't catch typos in asset names — you'll get nil at runtime. Some teams generate a .swift enum from asset names to make it compile-time-safe.
    • Don't drop loose images into the project navigator. Always go through the asset catalog — you lose @2x/@3x scaling, dark-mode variants, and the asset compiler optimization otherwise.
    • Trim before committing. Assets bloat app size faster than anything else. Use WebP or HEIC for photo-style images; SVG/PDF for vectors that need to scale.
