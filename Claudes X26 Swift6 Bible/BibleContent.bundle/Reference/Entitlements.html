Reference/Entitlements

  1. Entitlements (MyApp.entitlements)

  2. Reference: capability gating file

  3. What this file is

  4. The .entitlements file is a property list listing the protected APIs your app is allowed to use. Where Info.plist declares what your app says about itself, the entitlements file declares what the OS lets it do. Code-signing checks that every entitlement claimed in this file matches the provisioning profile signed into the binary — if they don't match, the app fails to launch.

  5. How entitlements come into existence

  6. You don't usually edit this file by hand. In Xcode, you toggle capabilities in the Signing & Capabilities tab of the target editor (the + Capability button at the top). Each capability you add writes corresponding entries into the .entitlements file and registers them with Apple's developer portal so your provisioning profile carries them.

  7. Common entitlements

  8. iCloud & CloudKit

    • com.apple.developer.icloud-services — CloudKit, CloudDocuments, CloudKeyValueStore.
    • com.apple.developer.icloud-container-identifiers — the iCloud container(s) your app accesses.
    • com.apple.developer.ubiquity-container-identifiers — for iCloud Drive document support.

  10. Push notifications

    • aps-environment — development or production.
    • com.apple.developer.usernotifications.communication — for Communication Notifications (richer message-style alerts).

  12. Sign in with Apple

    • com.apple.developer.applesignin — required if you offer "Sign in with Apple."

  14. App Groups & Keychain Sharing

    • com.apple.security.application-groups — lets your app + extensions (widgets, share extensions) share UserDefaults, files, and Core Data stores.
    • keychain-access-groups — lets multiple apps from the same team share Keychain items.

  16. HealthKit, HomeKit, Apple Pay, Maps

    • com.apple.developer.healthkit — HealthKit access. Pairs with NSHealthShareUsageDescription in Info.plist.
    • com.apple.developer.homekit — HomeKit accessory control.
    • com.apple.developer.in-app-payments — Apple Pay merchant IDs.
    • com.apple.developer.maps — access to MapKit's expanded routing/transit data.

  18. App Sandbox (macOS App Store)

  19. Mac App Store apps must be sandboxed. Toggle com.apple.security.app-sandbox on, then add per-feature sandbox entitlements:

    • com.apple.security.network.client, com.apple.security.network.server
    • com.apple.security.files.user-selected.read-write — access to files via NSOpenPanel/NSSavePanel
    • com.apple.security.files.downloads.read-write
    • com.apple.security.assets.music.read-write
    • com.apple.security.automation.apple-events — sending Apple Events (for AppleScript automation)
    • com.apple.security.temporary-exception.apple-events — targeting specific apps (Music, Mail, Finder)

  21. Hardened Runtime (notarized Mac apps, not sandboxed)

  22. Self-distributed Mac apps don't need the sandbox, but they DO need Hardened Runtime to be notarized. Hardened Runtime is configurable via separate entitlements:

    • com.apple.security.cs.allow-jit — allow Just-In-Time compilation
    • com.apple.security.cs.disable-library-validation — load unsigned plugins
    • com.apple.security.cs.allow-dyld-environment-variables — allow DYLD_* env vars

  24. The pairing rule (Info.plist + Entitlements)

  25. Many Apple capabilities require both an entitlement (machine permission) AND an Info.plist usage-description string (human explanation):

    • HealthKit → com.apple.developer.healthkit entitlement + NSHealthShareUsageDescription + NSHealthUpdateUsageDescription in Info.plist.
    • Apple Pay → com.apple.developer.in-app-payments entitlement (no Info.plist string needed).
    • Sign in with Apple → com.apple.developer.applesignin entitlement only.
  27. If your app crashes immediately on launch with a code-signing error, 80% of the time it's an entitlement that's claimed in .entitlements but missing from your provisioning profile. Re-generate the profile after adding the capability in Xcode — the auto-managed flow usually handles this, but a manual profile means you have to re-download from the developer portal.
