  6. 1. North Star — The Final Product

  7. 1.1 — Final shipping format is EPUB.

    One source of truth, packaged as EPUB for multi-venue distribution.

  8. 1.2 — Target publishing venues: Apple Books, Amazon Kindle, any other venue that accepts EPUB.
  9. 1.3 — During-making authoring environment is a vault of HTML rich-text documents.

    Not plain markdown. HTML with fully-functioning hyperlinks between Books, and multimedia (images, and as appropriate video/audio) embedded directly. Vault style similar in concept to Ulysses / Scrivener / Obsidian — but the vault is plain HTML so no tool lock-in and the files double as the EPUB source.

  10. 1.4 — Two-layer model.

    Source layer: HTML vault. Publishing layer: EPUB built by packaging the HTML vault — zipping XHTML + CSS + media + manifest. No format translation at publish time; EPUB is just packaging.

  17. 3. Architecture Decisions Still Open

  18. 3.1 Part numbering.

    The new Swift section pushes everything down. Options: (a) Swift becomes Part I, existing Part I-V slide down to Parts II-VI. (b) Swift is Volume Zero / Prelude, unnumbered, and existing Parts keep their numbers. (c) Total renumber into a new scheme.

  19. 3.2 Shared assets location.

    Cover image, stylesheets, embedded fonts (FiraCode Nerd Font), hero graphics, figures used by multiple Books — where do they live? Options: top-level _shared/ folder (proposed in 2.2); top-level _assets/; or each Book gets a copy.

  20. 3.3 Cross-linking convention.

    How one Book refers to another. Options: plain relative <a href="../Book-B/Book-B.html">; CSS-classed links (class="see-also", class="defined-in", class="cross-ref"); or a data-attribute system that lets us style/index them.

  21. 3.4 Multimedia scope.

    Images are given (figures/ already exists). What else? Options: (a) images only; (b) images + video (short MP4s of Xcode demos, SwiftUI animation clips); (c) images + video + audio. EPUB supports all of them but Kindle has uneven video support — this affects what we include in the Kindle build vs. the Apple Books build.

  22. 3.5 — Vault viewer: Obsidian now (open Bible root as vault), Claude Web Wrapper derivative later if/when volume justifies it.

    Confirmed 2026-04-20. Obsidian gives file tree + search for free while content is built out; clicking an HTML file launches it in the browser where line-numbered proofing already works. The custom reader app is deferred until the vault is large enough that the file-tree-plus-browser flow starts to feel clunky.

  23. 3.6 Entry format for Swift lexicon Books.

    Still open from the Swift section mapping doc — candidate fields: headword, category tag, plain-English definition, tiny example, related words, double-citation sources.

  32. 5. Proposed Phases

  33. Draft order of work — not locked. Each phase is a mappable milestone.

  34. Phase 0 — Architecture Setup

    Create the vault folder tree. Build the shared CSS stylesheet (print-ready + screen-reading + amber-proofing modes). Lock the hyperlink convention. Set up embedded FiraCode Nerd Font. Draft the EPUB packaging script. No book content written yet.

  35. Phase 1 — Migration

    Convert the 22 existing .md chapters to HTML in their new Book folders. Wire up internal cross-references. Move figures/media into per-Book folders. Keep .md files as an archive until migration is verified.

  36. Phase 2 — Swift Section Build (26 Books A–Z)

    Build out Part I: The Swift Language. Entry format locked first, then sample Book (likely Book A) proofed HTML-style, then the rest. Double-citation sources for each entry.

  37. Phase 3 — Content Gaps & New Books

    Chapter 16 proofing, Chapter 2 rebuild, LockBox build-process docs (Books 13–15), Web Wrapper appendix success-path rewrite. Any new Books identified during roadmap proofing.

  38. Phase 4 — Multimedia Pass

    Where appropriate, add video/audio/interactive media to existing Books. Scope set by answer to 3.4.

  39. Phase 5 — Proofing Sweep

    Full-book HTML proofing using the line-numbered technique. Double-citation verification pass. Accessibility pass (alt text, reading order, semantic markup). Style consistency.

  40. Phase 6 — EPUB Build & Publish

    Final EPUB v1.0 package. Apple Books submission. Amazon Kindle (KDP) submission. Any additional venues. Launch.

  41. Phase 7 — Living Book

    Bible is versioned. New Swift/SwiftUI/Xcode releases update relevant Books. Re-package and re-submit on release cadence.

  43. 6. Companion-App Constellation (context, not roadmap items)

  44. 6.1 Already shipping — QuickNote, Claude’s Web Wrapper, LockBox. Each has (or will have) a dedicated Appendix in the Bible showing the build-along.
