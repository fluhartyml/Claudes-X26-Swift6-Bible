Appendix B: CryoKit (Shared Package Source Tour)

Status: LIVE as a Swift package. CryoKit is a shared internal Swift package that both CryoTunes Player (Appendix E) and Tally Matrix Clock (Appendix F) depend on. This appendix is the book's one example of cross-app shared code — a curated package serving multiple production apps.

Source tour only. CryoKit is deliberately bare-bones — no fonts, no aesthetic controls, no dependencies of its own. Its scope is data and logic, nothing else. That discipline is the lesson: a shared package earns its keep by staying narrow.

What CryoKit contains

  • Radio station data structures used by CryoTunes Player
  • Station selection logic shared between the apps that depend on it
  • Deliberately no UI code, no fonts, no color ramps — pure data and logic

Why CryoKit is a separate package

CryoTunes Player streams Apple Music radio stations. Tally Matrix Clock surfaces a tiny station widget. Both apps need the same station data. Copying the data into each app means updates have to be made twice and kept in sync manually. A shared package solves that — the station data lives in one place, both apps import it via Swift Package Manager, and updates propagate naturally.

That's the pattern worth studying: shared code graduates into a package when two or more apps need the same behavior. Not before. Premature packaging creates overhead without benefit; late packaging forces painful deduplication.

Browse the source

  • Repo: github.com/fluhartyml/CryoKit
  • Wiki: github.com/fluhartyml/CryoKit/wiki
  • Added to an Xcode project via File → Add Package Dependencies → paste the repo URL

How the apps depend on it

  • Appendix E (CryoTunes Player) imports CryoKit for its radio station catalog and playback logic.
  • Appendix F (Tally Matrix Clock) imports CryoKit for the station widget that sits alongside its clock + weather display.

The bare-bones rule

CryoKit stays deliberately narrow — no fonts, no styling, no view code. That's the rule: a shared package exports data and logic, not aesthetics. Each consuming app styles its own UI with its own design system. If CryoKit shipped fonts, two apps would be locked into the same look whether they wanted it or not. Keeping the package bare-bones keeps the consuming apps free.
