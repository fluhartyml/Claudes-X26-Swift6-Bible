Part III · Book 09

  1. Chapter 9: Text and TextField

  2. Text View Basics

  3. Text is the workhorse for displaying read-only strings in SwiftUI.

  4. Text("Hello, world")
Text(verbatim: "Hello, world") // skips localization lookup

  5. Use verbatim: when you have a string that should never be localized (user-generated content, codes, etc.).

  6. String Interpolation

  7. Standard Swift interpolation works inside Text:

  8. Text("Score: \(score)")
Text("Price: \(price, format: .currency(code: "USD"))")
Text("Date: \(date, format: .dateTime.month().day())")
Text("Progress: \(value, format: .percent)")

  9. The format: parameter is the modern way. It handles localization automatically.

  10. Markdown Support

  11. Text parses inline Markdown out of the box when you use a string literal:

  12. Text("This is **bold** and this is *italic*")
Text("Visit [Apple](https://apple.com)")
Text("Use `code` formatting")
Text("~~Strikethrough~~ text")

  13. Watch out: Markdown parsing only works with string literals or LocalizedStringKey. If you pass a String variable, it renders as plain text:

  14. let message = "This is **not bold**"
Text(message) // plain text, no markdown

// Fix: explicitly use LocalizedStringKey or AttributedString
Text(LocalizedStringKey(message)) // now it parses markdown

  15. Concatenating Text Views

  16. You can combine Text views with +:

  17. Text("Name: ").bold() + Text(userName).foregroundStyle(.secondary)

  18. This is the only way to apply different styles to parts of the same line of text without using AttributedString.

  20. Text Modifiers

  21. Font

  22. Text("Title").font(.title)
Text("Body").font(.body)
Text("Caption").font(.caption)
Text("Custom").font(.system(size: 24, weight: .bold, design: .rounded))
Text("Custom").font(.system(.title, design: .monospaced))

  23. Built-in dynamic type sizes: .largeTitle, .title, .title2, .title3, .headline, .subheadline, .body, .callout, .footnote, .caption, .caption2.

  24. Always prefer dynamic type over fixed sizes. It respects the user's accessibility settings.

  25. Color and Style

  26. Text("Styled").foregroundStyle(.primary)
Text("Tinted").foregroundStyle(.blue)
Text("Gradient").foregroundStyle(.linearGradient(
    colors: [.blue, .purple],
    startPoint: .leading,
    endPoint: .trailing
))

  27. foregroundStyle replaced foregroundColor (now deprecated). Use foregroundStyle.

  28. Weight and Emphasis

  29. Text("Bold").bold()
Text("Italic").italic()
Text("Both").bold().italic()
Text("Light").fontWeight(.light)
Text("Heavy").fontWeight(.heavy)
Text("Wide").fontWidth(.expanded)

  30. Spacing: Kerning vs Tracking

  31. Text("KERNING").kerning(2)   // adjusts space between character pairs
Text("TRACKING").tracking(2) // uniform spacing added to every character

  32. Practical difference: kerning respects ligatures and pair-specific adjustments. tracking is uniform. For most UI work, tracking is what you want. For body text, kerning is more typographically correct.

  33. Watch out: You cannot use both on the same Text. If you apply both, kerning is ignored.

  34. Line Limits and Truncation

  35. Text("Long text here...")
    .lineLimit(2)
    .truncationMode(.tail) // .head, .middle, .tail

Text("Flexible")
    .lineLimit(1...5) // minimum 1 line, expand up to 5

  36. Baseline and Spacing

  37. Text("Shifted").baselineOffset(5) // positive = up, negative = down
Text("Spaced").lineSpacing(8)     // extra space between lines

  38. Text Case

  39. Text("hello").textCase(.uppercase) // "HELLO"
Text("HELLO").textCase(.lowercase) // "hello"
Text("hello").textCase(nil)        // no transformation

  40. Selection

  41. Make text selectable by the user:

  42. Text("Copy this").textSelection(.enabled)

  44. TextField

  45. TextField is for single-line text input.

  46. @State private var name = ""

TextField("Enter your name", text: $name)

  47. The first argument is the prompt/placeholder text.

  48. Prompt Parameter

  49. For more control over the placeholder:

  50. TextField(text: $name, prompt: Text("Full name").foregroundStyle(.tertiary)) {
    Text("Name") // this is the label, used by accessibility
}

  51. Axis Parameter (Expanding TextField)

  52. A TextField can expand vertically with the axis parameter:

  53. TextField("Write something", text: $bio, axis: .vertical)
    .lineLimit(3...6) // starts at 3 lines, grows to 6

  54. This gives you a text field that behaves like a growing text area. It starts compact and expands as the user types. Combine with lineLimit to control the range.

  55. Watch out: Without lineLimit, an axis: .vertical text field will grow without bound.

  56. Keyboard Types

  57. TextField("Email", text: $email)
    .keyboardType(.emailAddress)

TextField("Phone", text: $phone)
    .keyboardType(.phonePad)

TextField("Number", text: $amount)
    .keyboardType(.decimalPad)

  58. Common types: .default, .asciiCapable, .numberPad, .decimalPad, .phonePad, .emailAddress, .URL, .webSearch.

  59. Autocapitalization

  60. TextField("Name", text: $name)
    .textInputAutocapitalization(.words)

  61. Options: .never, .characters, .words, .sentences.

  62. Watch out: .textInputAutocapitalization replaced the old .autocapitalization modifier. Use the new one.

  63. Autocorrection

  64. TextField("Username", text: $username)
    .autocorrectionDisabled(true)

  65. Text Content Type

  66. Tells the system what kind of data this field collects, enabling autofill:

  67. TextField("Email", text: $email)
    .textContentType(.emailAddress)

TextField("Street", text: $street)
    .textContentType(.streetAddressLine1)

  68. onSubmit

  69. Fires when the user taps Return:

  70. TextField("Search", text: $query)
    .onSubmit {
        performSearch()
    }

  71. You can also set the return key label:

  72. TextField("Search", text: $query)
    .submitLabel(.search) // .done, .go, .join, .next, .return, .search, .send
    .onSubmit { performSearch() }

  73. Formatting Input

  74. Use format: for typed input that auto-formats:

  75. @State private var amount: Double = 0

TextField("Amount", value: $amount, format: .currency(code: "USD"))
TextField("Percent", value: $percent, format: .percent)
TextField("Count", value: $count, format: .number)

  76. Watch out: The binding updates only when the user commits (presses Return), not on every keystroke.

  78. SecureField

  79. For password entry. Characters are hidden.

  80. @State private var password = ""

SecureField("Password", text: $password)
    .textContentType(.password)
    .onSubmit { authenticate() }

  81. SecureField supports the same modifiers as TextField: .onSubmit, .submitLabel, .textContentType, .textInputAutocapitalization, etc.

  83. @FocusState

  84. Manage which field has keyboard focus programmatically.

  85. Boolean Focus (Single Field)

  86. @FocusState private var isNameFocused: Bool

TextField("Name", text: $name)
    .focused($isNameFocused)

Button("Focus Name") {
    isNameFocused = true
}

  87. Enum Focus (Multiple Fields)

  88. enum Field: Hashable {
    case firstName, lastName, email
}

@FocusState private var focusedField: Field?

Form {
    TextField("First", text: $first)
        .focused($focusedField, equals: .firstName)
        .onSubmit { focusedField = .lastName }

    TextField("Last", text: $last)
        .focused($focusedField, equals: .lastName)
        .onSubmit { focusedField = .email }

    TextField("Email", text: $email)
        .focused($focusedField, equals: .email)
        .onSubmit { submit() }
}
.onAppear { focusedField = .firstName }

  89. This creates a tab-through flow: Return moves focus to the next field.

  90. Dismiss Keyboard

  91. // Set focus to nil to dismiss keyboard
focusedField = nil

  92. Watch out: @FocusState must be Optional when used with an enum (the nil state means nothing is focused). The boolean variant is non-optional.

  93. Watch out: Setting focus in onAppear can be unreliable on first launch. If you need guaranteed focus on appear, wrap it in a brief task delay:

  94. .task {
    try? await Task.sleep(for: .milliseconds(500))
    focusedField = .firstName
}

  96. Common Patterns

  97. Search Bar

  98. @State private var searchText = ""
@FocusState private var searchFocused: Bool

HStack {
    Image(systemName: "magnifyingglass")
        .foregroundStyle(.secondary)

    TextField("Search", text: $searchText)
        .focused($searchFocused)
        .onSubmit { runSearch() }
        .submitLabel(.search)

    if !searchText.isEmpty {
        Button {
            searchText = ""
            searchFocused = true
        } label: {
            Image(systemName: "xmark.circle.fill")
                .foregroundStyle(.secondary)
        }
        .buttonStyle(.plain)
    }
}
.padding(8)
.background(.quaternary, in: .rect(cornerRadius: 10))

  99. Form with Validation

  100. @State private var email = ""
@FocusState private var emailFocused: Bool

var emailIsValid: Bool {
    email.contains("@") && email.contains(".")
}

TextField("Email", text: $email)
    .focused($emailFocused)
    .keyboardType(.emailAddress)
    .textContentType(.emailAddress)
    .textInputAutocapitalization(.never)
    .autocorrectionDisabled()
    .overlay(alignment: .trailing) {
        if !email.isEmpty && !emailFocused {
            Image(systemName: emailIsValid ? "checkmark.circle" : "exclamationmark.circle")
                .foregroundStyle(emailIsValid ? .green : .red)
                .padding(.trailing, 8)
        }
    }

  102. Quick Reference

  103. | Modifier | Purpose | |---|---| | .font() | Set text font | | .foregroundStyle() | Set text color/style | | .bold() / .italic() | Weight and emphasis | | .kerning() / .tracking() | Letter spacing | | .lineLimit() | Constrain line count | | .textCase() | Force upper/lower case | | .textSelection() | Enable copy | | .focused() | Bind to @FocusState | | .onSubmit {} | Handle Return key | | .submitLabel() | Return key appearance | | .keyboardType() | Keyboard layout | | .textInputAutocapitalization() | Cap behavior | | .textContentType() | Autofill hint | | .autocorrectionDisabled() | Kill autocorrect |
