  1. Chapter 06: Controls — Buttons, Toggles, and Pickers

  2. Button

  3. Basic Button

  4. Button("Save") {
    saveDocument()
}

Button(action: saveDocument) {
    Label("Save", systemImage: "square.and.arrow.down")
}

  5. Button Roles

  6. Button("Delete", role: .destructive) {
    deleteItem()
}

Button("Cancel", role: .cancel) {
    dismiss()
}

  7. .destructive renders in red. .cancel tells the system this dismisses without action (used in dialogs and confirmations).

  8. Button with Custom Label

  9. Button {
    startDownload()
} label: {
    HStack {
        Image(systemName: "arrow.down.circle.fill")
            .font(.title2)
        VStack(alignment: .leading) {
            Text("Download").font(.headline)
            Text("42 MB").font(.caption).foregroundStyle(.secondary)
        }
    }
    .padding()
    .background(.blue.opacity(0.1), in: RoundedRectangle(cornerRadius: 12))
}

  11. Built-in Button Styles

  12. Button("Bordered") { }
    .buttonStyle(.bordered)

Button("Bordered Prominent") { }
    .buttonStyle(.borderedProminent)

Button("Borderless") { }
    .buttonStyle(.borderless)

Button("Plain") { }
    .buttonStyle(.plain)

  13. | Style | Look | |-------|------| | .automatic | Platform default | | .bordered | Tinted background, rounded shape | | .borderedProminent | Filled with tint color, white text | | .borderless | Text only, no background | | .plain | No styling at all — just the label |

  14. Tint Control

  15. Button("Accept") { }
    .buttonStyle(.borderedProminent)
    .tint(.green)

  17. Custom ButtonStyle

  18. Implement the ButtonStyle protocol for a reusable button appearance.

  19. struct ScaleButtonStyle: ButtonStyle {
    func makeBody(configuration: Configuration) -> some View {
        configuration.label
            .scaleEffect(configuration.isPressed ? 0.95 : 1.0)
            .opacity(configuration.isPressed ? 0.8 : 1.0)
            .animation(.easeInOut(duration: 0.15), value: configuration.isPressed)
    }
}

// Usage
Button("Tap Me") { }
    .buttonStyle(ScaleButtonStyle())

  20. PrimitiveButtonStyle

  21. For full control over when the action fires (e.g., requiring a long press).

  22. struct LongPressButtonStyle: PrimitiveButtonStyle {
    func makeBody(configuration: Configuration) -> some View {
        configuration.label
            .onLongPressGesture {
                configuration.trigger()
            }
    }
}

  23. Watch Out

    • ButtonStyle gives you isPressed state but the system handles the tap. PrimitiveButtonStyle gives you full control but you must call configuration.trigger() yourself.
    • buttonStyle applies to all buttons in the subtree. Set it on a container to style every button inside it at once.

  26. Toggle

  27. Basic Toggle

  28. @State private var isEnabled = false

Toggle("Notifications", isOn: $isEnabled)

Toggle(isOn: $isEnabled) {
    Label("Notifications", systemImage: "bell")
}

  29. Toggle Styles

  30. Toggle("Option", isOn: $value)
    .toggleStyle(.switch)      // the default iOS switch

Toggle("Option", isOn: $value)
    .toggleStyle(.button)      // renders as a pressable button

Toggle("Option", isOn: $value)
    .toggleStyle(.checkbox)    // macOS only

  31. Tint

  32. Toggle("Dark Mode", isOn: $darkMode)
    .tint(.purple)

  33. Watch Out

    • .toggleStyle(.checkbox) is macOS only. On iOS it falls back to a switch.
    • Toggles in a List automatically get trailing alignment. Outside a List, they default to leading label with trailing switch.

  36. Picker

  37. Menu Picker (Default on iOS)

  38. @State private var selectedFlavor = "Chocolate"
let flavors = ["Chocolate", "Vanilla", "Strawberry"]

Picker("Flavor", selection: $selectedFlavor) {
    ForEach(flavors, id: \.self) { flavor in
        Text(flavor)
    }
}

  39. Segmented

  40. Picker("View", selection: $viewMode) {
    Text("Grid").tag(ViewMode.grid)
    Text("List").tag(ViewMode.list)
    Text("Gallery").tag(ViewMode.gallery)
}
.pickerStyle(.segmented)

  41. Wheel

  42. Picker("Weight", selection: $weight) {
    ForEach(100...300, id: \.self) { w in
        Text("\(w) lbs").tag(w)
    }
}
.pickerStyle(.wheel)

  43. Inline (Expands in List)

  44. Picker("Category", selection: $selectedCategory) {
    ForEach(categories) { cat in
        Text(cat.name).tag(cat)
    }
}
.pickerStyle(.inline)

  45. Navigation Link Picker

  46. Inside a NavigationStack, this pushes a full selection list.

  47. Picker("Country", selection: $selectedCountry) {
    ForEach(countries) { country in
        Text(country.name).tag(country)
    }
}
.pickerStyle(.navigationLink)

  48. Watch Out — The #1 Picker Bug

  49. Every picker option must have a .tag() that matches the exact type of the selection binding. If your selection is Optional<Item>, your tags must be Optional<Item> too — not plain Item.

  50. // WRONG — selection is Item?, but tags are Item
@State private var selected: Item?
Picker("Pick", selection: $selected) {
    Text("None").tag(nil)          // nil has no type context
    ForEach(items) { item in
        Text(item.name).tag(item)  // Item, not Item?
    }
}

// RIGHT
Picker("Pick", selection: $selected) {
    Text("None").tag(nil as Item?)
    ForEach(items) { item in
        Text(item.name).tag(item as Item?)
    }
}

  51. This is the number one source of "my picker doesn't update" bugs. If the picker does not respond to taps, check tag types first.

  53. Stepper

  54. @State private var quantity = 1

Stepper("Quantity: \(quantity)", value: $quantity, in: 1...99)

Stepper(value: $zoomLevel, in: 0.5...3.0, step: 0.25) {
    Text("Zoom: \(zoomLevel, specifier: "%.2f")x")
}

  55. Manual Stepper

  56. Stepper("Custom") {
    increment()
} onDecrement: {
    decrement()
}

  58. Slider

  59. @State private var volume: Double = 0.5

Slider(value: $volume)

Slider(value: $volume, in: 0...100, step: 5) {
    Text("Volume")
} minimumValueLabel: {
    Image(systemName: "speaker")
} maximumValueLabel: {
    Image(systemName: "speaker.wave.3")
}

  60. Watch Out

    • Slider labels are hidden by default in many contexts. Use an explicit Text above the slider if you need the user to see the label.
    • step only snaps during dragging. Setting step: 5 with range 0...100 means the user drags in increments of 5.

  63. DatePicker

  64. @State private var date = Date()

DatePicker("Birthday", selection: $date, displayedComponents: .date)

DatePicker("Alarm", selection: $date, displayedComponents: .hourAndMinute)

DatePicker("Event", selection: $date,
           in: Date()...,   // only future dates
           displayedComponents: [.date, .hourAndMinute])

  65. Styles

  66. .datePickerStyle(.graphical)    // calendar grid
.datePickerStyle(.wheel)        // spinning wheels
.datePickerStyle(.compact)      // tappable pill that expands
.datePickerStyle(.automatic)    // platform decides

  67. Watch Out

    • .graphical takes up a lot of space. Works well in a Form or sheet, but can break layouts in tight spaces.
    • The in: parameter sets the selectable range. Date()... means today onward. ...Date() means up to today.

  70. ColorPicker

  71. @State private var color: Color = .blue

ColorPicker("Theme Color", selection: $color)

ColorPicker("Background", selection: $color, supportsOpacity: false)

  72. supportsOpacity: false hides the alpha slider.

  74. Disabled States

  75. Button("Submit") { submit() }
    .disabled(formIsInvalid)

Toggle("Feature", isOn: $feature)
    .disabled(!isPremiumUser)

  76. .disabled(true) grays out the control and blocks interaction. It propagates down the view tree — disable a VStack and every control inside is disabled.

  77. Reading Disabled State in Custom Views

  78. struct MyControl: View {
    @Environment(\.isEnabled) var isEnabled

    var body: some View {
        Text("Status: \(isEnabled ? "Active" : "Disabled")")
            .foregroundStyle(isEnabled ? .primary : .secondary)
    }
}

  80. Labels and Accessibility

  81. Label

  82. Label pairs an icon with text. Many controls accept a Label as content.

  83. Label("Downloads", systemImage: "arrow.down.circle")
Label("Warning", systemImage: "exclamationmark.triangle")
    .foregroundStyle(.orange)

  84. Label Styles

  85. Label("Item", systemImage: "star")
    .labelStyle(.titleAndIcon)  // both text and icon
    .labelStyle(.titleOnly)     // text only
    .labelStyle(.iconOnly)      // icon only

  86. Accessibility Labels

  87. Button(action: deleteItem) {
    Image(systemName: "trash")
}
.accessibilityLabel("Delete item")

Slider(value: $brightness)
    .accessibilityLabel("Screen brightness")
    .accessibilityValue("\(Int(brightness))%")

  88. Watch Out

    • Icon-only buttons must have an .accessibilityLabel. VoiceOver users hear nothing otherwise.
    • Label does not always show both icon and text. In toolbars and menus, the system may choose to show only one. Use .labelStyle(.titleAndIcon) to force both.

  91. Practical Tips

    1. Use .borderedProminent sparingly. One prominent button per screen for the primary action. Everything else gets .bordered or .borderless.
    1. Picker tag type matching is the most common SwiftUI bug. When your picker ignores taps, check tag types first.
    1. Stepper is great for small ranges (1-20). For large ranges, use a Slider or a text field with validation.
    1. Group related controls in a Form. Controls inside a Form automatically get platform-appropriate styling — grouped rows on iOS, aligned labels on macOS.
    1. Test with Dynamic Type. At the largest accessibility sizes, labels wrap and controls reflow. Make sure nothing gets clipped.
    1. 18pt minimum font height. Per project standards, all text in controls should meet this minimum for iPad readability.
