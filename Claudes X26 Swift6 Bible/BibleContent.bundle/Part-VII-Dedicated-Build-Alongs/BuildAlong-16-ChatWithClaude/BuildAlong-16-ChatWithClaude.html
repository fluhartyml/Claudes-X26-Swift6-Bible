Build-Along 16: ChatWithClaude

Status: planning phase as of 2026-04-24. App not yet scaffolded. Walkthrough fills in once v1.0 scaffolds; expanded planning notes below.

MVP concept: a BYO-Anthropic-API-key chat workbench for Claude. Readers bring their own API key, saved securely in Keychain, and use the app to hold multiple long-running conversations with Claude across models (Opus / Sonnet / Haiku), with streaming responses, markdown rendering, token-cost tracking, and conversation export. Differentiates from Anthropic's official Claude app by being a workbench for the reader's own API usage — not a subscription client.

What ChatWithClaude will teach

  • Anthropic API — the /v1/messages streaming endpoint, server-sent events parsing, message turn structure, system prompts.
  • URLSession streaming — URLSession.bytes(for:) + AsyncSequence for incremental response rendering. Readers see a SwiftUI chat bubble fill in token by token.
  • Keychain — storing the API key via the Security framework, not UserDefaults. Real credential hygiene.
  • SwiftData — persisting conversations, messages, and cost tracking between sessions.
  • Prompt caching — marking cache-eligible content with cache_control; readers see the 90% cost reduction on cached context, a feature most third-party clients skip. This is the differentiating teaching moment of the chapter.
  • Markdown rendering — SwiftUI AttributedString rendering of Claude's markdown responses (headers, lists, code blocks with syntax highlighting, links).
  • PDFKit (supporting) — export a conversation to PDF for sharing (reuses the skill from Build-Along 11 ReportMaker).
  • Signature layer: MVP + Under the Hood + About & Send Feedback.
  • Distribution: App Store — BYO-API-key workbenches for LLMs are an established category (TypingMind, Msty) and Apple doesn't ship one. Clears 2.2 as a real utility.

Core flows

  • First launch — prompt for Anthropic API key, save to Keychain, test with a minimal claude-haiku-4-5 ping before exiting the setup screen.
  • New conversation — pick a model (default Sonnet 4.6), optional system prompt preset, start chatting. Responses stream in real time.
  • Conversation list — reverse-chronological; each conversation shows title, model used, message count, total token cost, last-updated timestamp.
  • Export — share a conversation as markdown or PDF via the share sheet.
  • Cost tracking — token counts tallied per message + conversation, priced against current rates, surfaced in the cost column. Readers see the price of their own usage.

Model selection

  • Opus 4.7 — flagship, hardest reasoning, slowest, most expensive.
  • Sonnet 4.6 — balanced default for most work.
  • Haiku 4.5 — fastest + cheapest for quick turn-around.

Readers see the cost + latency tradeoff directly in-app as they switch models mid-conversation.

Why ChatWithClaude passes App Review on merit

BYO-API-key LLM clients are a mature category. Apple doesn't ship one. User brings their own account + key; the app never collects or transmits anyone else's data. Prompt caching as a built-in feature is genuinely differentiating — most competing clients don't implement it. Clean SwiftUI rendering of streamed markdown over the Anthropic API is the teaching payoff for the chapter.
