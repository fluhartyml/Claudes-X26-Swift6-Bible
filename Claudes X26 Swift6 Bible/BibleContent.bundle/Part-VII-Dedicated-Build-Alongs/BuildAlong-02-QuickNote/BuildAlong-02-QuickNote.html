Appendices/Appendix C QuickNote

  1. Appendix: QuickNote — From Brainstorm to App Store

  2. This appendix walks through the complete lifecycle of an iOS app — from the first brainstorm to App Store submission, rejection, and resubmission. QuickNote by Claude is a real app you can download. Every decision, every mistake, and every fix is documented here because that's how software actually gets built.

  4. The Brainstorm

  5. QuickNote started as the Swift reference's first example app. Dual purpose: a teaching vehicle for the book AND a real App Store utility. Not a throwaway exercise — a published app that readers can download and compare against the source code.

  6. v1.0 spec:

    • Text notes with titles and dates
    • SwiftData persistence
    • iCloud sync
    • Share sheet
    • Multiplatform (iOS, iPadOS, macOS)
    • Pre-dating: place notes on your future timeline (doctor visits, grocery runs)
    • Sort by dateCreated descending, not dateModified

  8. v2.0 deferred: Camera, document scanner, OCR (VisionKit chapters). Each Swift Reference example app gets revisited when later chapters add relevant features.

  10. The Data Model

  11. Four properties, in this order everywhere:

  12. @Model
final class Note {
    var title: String
    var dateCreated: Date
    var body: String
    var dateModified: Date

    init(title: String = "", dateCreated: Date = .now,
         body: String = "", dateModified: Date = .now) {
        self.title = title
        self.dateCreated = dateCreated
        self.body = body
        self.dateModified = dateModified
    }
}

  13. dateCreated is user-editable — the DatePicker lets you pre-date or back-date. dateModified updates automatically via .onChange modifiers on the title, date, and body fields.

  15. Build Errors That Taught Something

  16. These aren't bugs — they're lessons. Each one appears in the relevant chapter earlier in this book and is collected here for reference.

  17. 1. Date.FormatStyle Properties Aren't Assignable

  18. What happened: Tried to build a custom date format using Date.FormatStyle property chaining. The properties are read-only — you can't assign to them.

  19. Fix: Use DateFormatter with a dateFormat string:

  20. let quickNoteDateFormatter: DateFormatter = {
    let f = DateFormatter()
    f.dateFormat = "yyyy MMM dd HHmm"
    f.locale = Locale(identifier: "en_US_POSIX")
    return f
}()

  21. See: Chapter 2, Date Formatting section.

  22. 2. DateFormatter Returns Title Case

  23. What happened: DateFormatter returns "Apr" not "APR". The design called for uppercase month abbreviations.

  24. Fix: .uppercased() on the formatted string.

  25. Text(quickNoteDateFormatter.string(from: note.dateCreated).uppercased())

  26. 3. NavigationSplitView Sidebar Truncation

  27. What happened: Long note titles got cut off in the sidebar because NavigationSplitView defaults to a narrow column width.

  28. Fix: .navigationSplitViewColumnWidth(min: 350, ideal: 380)

  29. See: Chapter 5, NavigationSplitView section.

  30. 4. TextField(axis: .vertical) Treats Return as Submit

  31. What happened: Used TextField(axis: .vertical) for the note body. Pressing Return exits the field instead of creating a new line.

  32. Fix: Use TextEditor for real multiline input. TextField is for single-line input, even with the vertical axis hint.

  33. See: Chapter 9 (Text & TextField) and Chapter 10 (TextEditor & AttributedString).

  34. 5. App Name Conflicts

  35. What happened: "QuickNote" was already taken on the App Store. Archive > Validate App caught it.

  36. Fix: Renamed to "QuickNote by Claude" in the Display Name field. The bundle ID stayed com.ClaudeX26Bible.QuickNote.

  37. 6. Missing App Icons Fail Validation

  38. What happened: Validate App failed because the asset catalog was missing the required icon sizes — 1024x1024 for iOS and all sizes for Mac.

  39. Fix: Add the icons to the asset catalog before archiving. Xcode won't warn you during development — only during validation.

  40. 7. Simulator Screenshots Don't Match App Store Sizes

  41. What happened: iPhone simulator screenshots are 1320x2868. App Store Connect requires 1284x2778 for the 6.5" display.

  42. Fix: Batch resize with sips:

  43. sips -z 2778 1284 screenshot.png --out screenshot.png

  44. 8. Export Compliance

  45. What happened: App Store Connect asks about encryption every time you select a build. For a notes app: "None of the algorithms mentioned above."

  46. 9. Privacy Policy URL Requires HTTPS

  47. What happened: Entered an http:// URL for the privacy policy. App Store Connect requires https://.

  48. Fix: Host the privacy policy on the app's GitHub wiki — GitHub provides HTTPS by default.

  49. 10. Do App Privacy + Pricing BEFORE Distribution Metadata

  50. What happened: Filled out the Distribution page first (screenshots, description, keywords). Hit Save. Nothing saved. No error message.

  51. Fix: Complete App Privacy and Pricing sections first. The Distribution page won't save until those are done. App Store Connect doesn't tell you this — it just silently fails.

  52. 11. App Store Connect Caches Errors

  53. What happened: Fixed an issue, went back to the submission page. The error was still showing.

  54. Fix: Refresh the browser page. App Store Connect caches aggressively.

  56. The Rejection

  57. QuickNote v1.0 was rejected under Guideline 4.2 — Design: Minimum Functionality.

  58. > The usefulness of the app is limited by the minimal functionality it currently provides. Specifically, the app primarily offers content for users to view or use, but there isn't enough of this content currently available in the app to make it useful to users.

  59. Translation: a notes app that just creates, edits, and deletes text notes isn't enough. Apple Notes already does that. What makes yours different?

  60. The Fix

  61. Three features added in response:

    1. Under the Hood tab — a source code browser that displays every Swift file in the app. Pop the hood, see the engine. The book (this book) is the Chilton's manual. The app is the car.
    1. Home screen widget — shows the 3 most recent notes. Built with WidgetKit (see Chapter 16). Uses SwiftData to read from the same store as the main app.
    1. Contact Developer — a feedback form that composes a mailto: URL with the user's message and device info. Same pattern used in CryoTunes Player.

  65. The app category was also changed from Utilities to Education to match the teaching purpose.

  66. The Resubmission

  67. App Review notes explained exactly what changed and why:

  68. > This app was previously rejected under Guideline 4.2 (Minimum Functionality). We have added the following features in response: > 1. Under the Hood tab — users can browse the app's own Swift source code > 2. Home screen widget — displays the 3 most recent notes > 3. Contact Developer — built-in feedback form > This app serves as a companion app for an iOS development book.

  69. Build number gotcha: The new build wasn't immediately available in App Store Connect after uploading from Xcode. There's a processing delay — sometimes a few minutes. If you rush to resubmit, you might grab the old build by accident. Wait for the new build number to appear in the Add Build dialog.

  71. The Reject-and-Resubmit Strategy

  72. Sometimes an app sits in "Waiting for Review" for days with no movement. This happened with Tally Matrix Clock — all rejection issues fixed, resubmitted, and then silence for two days.

  73. The fix: Developer Reject your own submission, then resubmit. This puts you into a fresh review queue.

  74. How to do it:

    1. Go to App Store Connect > Your App > App Review
    2. Click on the submission that's stuck
    3. Scroll to the bottom — click "Cancel Submission"
    4. Confirm the cancellation
    5. Status changes from "Waiting for Review" → "Processing" → "Developer Rejected"
    6. Now go to Distribution, attach the new build, and submit again

  76. You'll land in a different reviewer's queue. This isn't gaming the system — Apple expects developers to use this when submissions stall.

  78. Lessons for All Example Apps

  79. Every Swift Reference example app will face these same hurdles. Plan for them:

    • Minimum functionality — a utility app needs more than just CRUD. Add a widget, add a Learn tab, add something that differentiates it.
    • Apple trademark compliance — don't use "Apple TV," "iPhone," or other Apple trademarks in your subtitle or promotional text. Use "your TV" or "your phone."
    • Screenshots must match App Store dimensions — simulator screenshots won't be the right size. Keep sips in your toolbox.
    • Build numbers must match — if your app embeds an extension (widget, share extension), both must have the same CFBundleVersion.
    • Do privacy and pricing first — before touching the Distribution page.
    • The review queue can stall — give it 48 hours, then reject and resubmit.

  83. v1.1 — The Widget Fix and Photo Attachments

  84. QuickNote v1.0 shipped with a broken widget. The widget showed "No Notes" even when the app had notes. Apple didn't catch it during review — they likely didn't test adding a note and checking the widget.

  85. The Bug

  86. The widget extension and the main app were using separate SwiftData stores. Each target created its own ModelContainer with no shared location:

  87. // v1.0 — BROKEN: each target gets its own private database
let config = ModelConfiguration(schema: schema, isStoredInMemoryOnly: false)

  88. The app wrote notes to its sandbox. The widget read from its own sandbox. Two empty rooms in the same building with no hallway between them.

  89. The Fix: App Groups + groupContainer

  90. Three things had to happen:

    1. Register an App Group on developer.apple.com — group.com.ClaudeX26Bible.QuickNote
    2. Enable App Groups in Xcode's Signing & Capabilities for both the app and widget extension targets
    3. Use groupContainer in the ModelConfiguration:
  92. // v1.1 — FIXED: both targets share the same database
let config = ModelConfiguration(
    schema: schema,
    groupContainer: .identifier("group.com.ClaudeX26Bible.QuickNote")
)

  93. That single parameter change — groupContainer: .identifier(...) — tells SwiftData to put the database in the shared App Group container instead of each target's private sandbox.

  94. The widget also needed WidgetCenter.shared.reloadAllTimelines() called after every note insert, edit, and delete so it refreshes immediately instead of waiting 30 minutes.

  95. Debugging the Fix

  96. The fix didn't work on the first try. Three attempts:

    1. Manual URL approach — built the container URL with FileManager.default.containerURL(forSecurityApplicationGroupIdentifier:) and appending(path:). SwiftData couldn't find the store.
    2. Named ModelConfiguration — added a name parameter. Caused a mismatch between app and widget.
    3. groupContainer: .identifier(...) — the API SwiftData actually provides for this exact use case. Worked immediately.

  98. The lesson: don't build your own paths when the framework has a dedicated API. The groupContainer parameter exists because Apple knew developers would need to share data between app and extension targets.

  99. Photo Attachments

  100. v1.1 also added photo attachments:

    • PhotosPicker from PhotosUI for selecting library images
    • Camera capture via UIImagePickerController wrapped in UIViewControllerRepresentable
    • Images stored as [Data] with @Attribute(.externalStorage) so binary data doesn't bloat the SwiftData row
    • Adaptive grid display above the note body with per-image delete buttons

  102. The model gained one property:

  103. @Attribute(.externalStorage) var imageData: [Data]

  104. This is a schema change — existing installs need to delete and reinstall to migrate. For a production app with users, you'd write a migration plan. For a v1.1 with one user (the developer testing it), a fresh install is fine.

  105. What v1.1 Teaches

    • App Groups are not optional for extensions — if your extension needs app data, configure this from day one
    • The developer portal and Xcode must agree — register the group online, then add it in Signing & Capabilities
    • Don't ship features you haven't tested end-to-end — the widget looked correct in the preview but was broken on the home screen
    • @Attribute(.externalStorage) keeps binary data efficient in SwiftData
    • PhotosPicker is the modern replacement for UIImagePickerController — except for camera, which still needs the UIKit bridge

  108. QuickNote by Claude is available on the App Store. Download it, open the Under the Hood tab, and follow along with this appendix.
