Build-Along 05: Outside

Status: planning phase as of 2026-04-24. App not yet scaffolded. Walkthrough fills in once v1.0 scaffolds; expanded planning notes below.

MVP concept: a weather-and-location diary. The app shows current weather and your location on a map, and lets you log a journal entry with a photo stamped against that moment — weather, place, time, photo, text — all preserved as one log entry you can browse later.

What Outside will teach

  • WeatherKit — current conditions for the user's coordinates, plus a forecast lookup for entry context.
  • CoreLocation — request permission, read device location, handle the authorization dance properly.
  • MapKit — render a map centered on the current location, drop a pin at each past log entry.
  • PhotosUI — pick a photo from the library, or snap a new one with the camera, attach it to the log entry.
  • SwiftData — persist log entries locally so the diary accumulates across sessions.
  • Signature layer: MVP feature + Under the Hood view + About & Send Feedback.
  • Distribution: App Store listing — full-fledged utility (weather diary is real, not a demo surface).

Per-entry data model (sketch)

@Model final class OutsideLog {
    var id: UUID
    var timestamp: Date
    var latitude: Double
    var longitude: Double
    var weatherSummary: String     // e.g. "72F, partly cloudy"
    var temperature: Double
    var condition: String          // Weather.symbolName
    var photoData: Data?           // optional attached photo
    var note: String               // user-typed log entry
}

UI shape (sketch)

  • Home tab — current weather card at top, map beneath with pins for past entries, "New Entry" button.
  • New Entry sheet — auto-fills weather + location + timestamp, PhotoPicker for image, TextEditor for the note, Save.
  • Log tab — reverse-chronological list of entries, tap to open full detail (photo, weather snapshot, location, text).

Current state

  • No screenshots, no icon
